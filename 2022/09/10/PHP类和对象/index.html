PHP——类和对象

发表于2022-09-10|更新于2022-10-14|编程代码
|阅读量:

面向对象

面向对象的编程(OOP)

软件工程的目标: 复用性 灵活性 扩展性.

面向对象的特点: 封装 继承 多态

对象(object)

对象就是具有一定功能和特征的事物.凡是能帮我们实现某种需求的事物的都是对象

a) 女朋友是对象.

b) 每一个学员都是讲师的教学对象.

c) 留言板项目每一条留言都是对象.

d) 商城项目. 把商品卖给用户.

​ 商品: 进货, 销售, 库存, 查看, 修改定价都是围绕着商品考虑的, 所以商品是对象.

​ 用户: 注册, 登录, 个人信息修改, 等等都是围绕着用户考虑问题,所以用户是对象.

项目开发中的 面向对象编程, 首先就是分析出要有哪些事物(对象)来完成整个项目的功能需求和业务逻辑.

然后,就是围绕着”对象”进行编码,实现需求功能, 最后让对象与对象之间互动(调用), 来完成业务流程的逻辑.

类(class)

具有相同特征和功能的对象的抽象 就是类

对象可以有很多个.比如,很多个学生对象, 很多个商品对象, 很多个用户对象.

很多个学生对象归纳为一个学生类.

很多个商品对象归纳为一个商品类.

很多个用户对象归纳为一个用户类.

我们在做 项目对象分析时, 其实就是在分析项目需要哪一类的对象.

把一些对象归为某个类,是因为这些对象会有相同的属性:

比如 商品类中, 每一个商品对象都会有 定价, 库存, 产地, 生产日期

把一些对象归为某个类, 还会因为对该类中每一个对象都会做相同或相似的操作.

如果 学生类中, 每一个学生对象都会有 听课, 写作业, 起立的动作.

因些, 描述某一个”对象类”包含哪些属性和哪些操作.

那么属于该类中的每一个对象都会有相同的属性名, 和相同的操作处理.

在PHP实际编码过程中, 首先描述某一类对象的属性和操作. 这称为 定义类.

随后就可以用”类的描述”来创建一个一个的实实在在的对象.并且往往给对象赋予不同的属性值.

对象和类之间的关系

对象参考类的结构,被创建出来.并且属性保存具体的数据.

它是具体的.

类是对所属对象的共有特性的描述, 可以说这是一个类型, 一个结构

它是抽象的

对象的抽象就是类

类的实体化(实例化)就是对象

构造方法

创建对象时, 自动触发构造方法, 可以在构造方法中快速给对象属性赋值

析构方法

对象被销毁时, 自动触发该方法.

  1. 对象是变量, 所以当变量被赋值时, 原来的对象被覆盖销毁.

  2. php代码结束时, 所有变量常量统统销毁, 对象是变量, 也被销毁.

  3. 使用unset销毁当前这个对象.

封装

使用修饰符修饰成员属性和成员方法,最大限度的隐藏对象的内部细节,保证对象的完整性和安全性

成员修饰符

public 公有的

类的成员属性和成员方法, 可以在类的 外部 读取和调用.

如果成员方法前没有任何修饰符, 则默认为 public

对象把变量和函数用大括号装在一起

即 对象 = {成员属性+成员方法} = {变量 + 函数}

强调的是一个整体性.

对象内部, 成员方法可以直接访问成员属性. 因此,方法之间传递数据非常方便.

private 私有的

类的成员属性和成员方法, 只能在类的 内部 读取和调用.

对象中的有些成员属性和方法, 不应该直接被外部访问( 把 public 换成 private ).

  1. 想要知道女生的名字, 不能直接拿人家身份证, 需要向她请求询问.

​ 女生回答(对象的成员方法), 其结果(返回值), 可以是真实姓名, 可以是假名字, 也可以拒绝告诉.

  1. 借给朋友钱, 不应该把银行卡和密码给对方, 让对方自已去取.

​ 而是应该隐藏细节, 只是把最终的钱借给他.

  1. 对象成员属性有合理的取值范围, 比如年龄. 如果外部直接赋值, 可能是1000岁 这样不合理的值.

​ 由对象自己提供专门的成员方法来设置,获取年龄.方法中进行必要的范围检查. 从而保证合理性.

有些成员只是被其它成员访问使用, 不需要被外部知道, 那就加上 private .

在使用一个类对象时, 我们只需要关心它的public的, 可以被外部访问的成员.

protected 受保护的

​ 受保护的成员属性在类的外部 不能直接访问

​ 受保护的成员方法在类的外部 不能直接调用

魔术方法

get()和set()

__get 获取对象中的非公有或不存在的成员属性时, 触发该方法 接受一个参数 属性名

__set 设置对象中的非公有或不存在的成员属性时, 触发该方法 接受两个参数 (属性名,属性值)

call()

__call() 调用对象中非公有或不存在的成员方法时, 触发该方法 接受两个参数 (函数名,参数列表)

isset()和unset()

__isset 判断对象的非公有或不存在的属性时, 触发该方法 接受一个参数 属性名

__unset 销毁对象的非公有或不存在的属性时, 触发该方法 接受一个参数 属性名

clone()

__clone 当对象被克隆时,触发该方法

invoke()

__invoke 当对象被当做函数调用时, 触发该方法

继承

就是别人有 但是我没有 这个时候需要使用继承 把它所有的东西都拿过来

父类中的 私有成员( private ) 在子类中也是无法访问的.

父类中使用 受保护的 ( protected ) 修饰成员, 使成员在子类中也可以访问.

本类 子类 类的外部
public Y Y Y
private Y N N
protected Y Y N

单继承

在php中只支持单继承

子类的重载

1.如果子类对父类的方法进行重写之后,实例化的对象在调用该方法的时候,会调用子类方法.

2.如果在子类的方法中想调用父类中的同名方法,这个需要使用 parent::方法名()

3.子类中重写成员方法的时候,要求参数个数和类型要一致,但是构造方法可以不一致

final 关键字

用来修饰类和成员方法, 用final修饰的类不能被继承,用final修饰的成员方法不能被重写

trait 关键字

与 class 相似, 但是用 trait 定义的结构不能实例化对象

通过在类中使用use 关键字，声明要组合的Trait名称

成员方法同名时的优先级别: 子类方法 => trait方法 => 父类方法

其他

static 静态修饰符

写在成员属性之前, 该成员称为静态成员属性

静态成员属性, 是静态的, 保存在类的定义内存中. 不管对象创建与销毁, 它都存在.

a) 静态成员属性

​ i. 静态修饰符(static)可以跟public protected private一起修饰成员属性,也可以单独来修饰成员属性

​ ii. 静态成员属性 是作为该类对象的一个共有属性存在

​ iii. 静态成员属性的访问方式

​ 1) 在类的内部 self::静态成员属性

​ 2) 在类的外部 类名::静态成员属性

b) 静态成员方法

​ i. 静态修饰符(static)可以跟public protected private一起修饰成员方法

​ ii. 静态成员方法中只能操作静态成员, 绝对不能在静态成员方法中使用$this

​ iii. 静态成员方法的访问形式

​ 1) 在类的内部 self::方法名

​ 2) 在类的外部 类名::方法名

常量 const

在类的内部定义常量, 只能使用 const

const和define的区别

​ i. const可以在类中定义常量 但是define不可以

​ ii. define赋值的时候可以使用变量进行赋值, 在函数的内部可以使用, const是不可以的

​ iii. define可以在流程控制中定义常量(if swtich) ,const是不可以的

单例模式

单例设计模式, 使某个类的对象仅允许实例化一次

1.不允许脚本在类的外部使用new关键字实例化对象.操作就是将构造方法和析构方法私有化

2.需要在类的内部声明一个公有的静态成员方法, 必须调用此方法 以返回实例的一个引用

3.声明静态成员属性存储新产生的对象

4.私有的__clone()方法防止克隆对象

1.__autoload

a) __autoload是一个魔术函数(不是一个魔术方法)

b) 当使用new 关键字实例化一个不存在的类的时候 ,继承一个不存在的类的时候自动触发

c) 接受一个参数 不存在的类名(字符串类型)

d) 作用就是自动加载类文件 (类的自动加载)

2.__toString

a) __toString是一个魔术方法

b) 触发场景 就是当在脚本中直接对对象进行字符串操作的时候 自动触发

c) 在方法中必须要返回一个字符串类型的值

d) 作用: 代替对象进行字符串的操作

3.serialize(串行化/序列化) 和 unserialize(反串行化/反序列化)

serialize 是一个函数.

作用是将一个数组或者是对象 转化成一个字符串 便于持久化保存

应用场景: 缓存 方便数据传输

unserialize 是一个函数

作用是将字符串还原成原来的数据,可能是数组也可能是对象

魔术方法

__sleep

是一个魔术方法

当使用serialize进行串行化对象的时候 自动执行

作用 是要筛选要保留的属性的名称.

注意点: 必须要返回一个数组.

__wakeup

是一个魔术方法

当使用unserialize反串行化一个对象的时候,会自动触发该魔术方法

作用 就是为了初始化反串行化出来的对象.

抽象类

abstract 修饰符

a). abstract用来修饰 类或者成员方法

b). 修饰的类不能实例化对象, 修饰的成员方法没有方法体

c). 子类继承抽象类 需要将抽象类中的所有抽象方法完全实现才能实例化对象,如果只实现了部分,那么该类仍然是一个抽象类 ,必须使用abstract来修饰

d). 抽象类就好比是一个不完整的设计

e). 如果一个类的内部 包含一个抽象方法,那么该类就是一个抽象类,必须要使用abstract来修饰

接口

a). 接口是一种特殊的类

b). 声明方式用interface关键字

c). 接口中只能存在两种类型的代码 常量 抽象方法(抽象方法无需使用abstract来修饰,默认全部都是抽象方法)

d). 子类在继承(实现)接口的时候 ,使用的关键字是 implements

e). 如果子类在实现接口的时候 ,将接口中所有的抽象方法完全实现,那么该类就是一个完整的设计,就能实例化对象,如果只实现部分方法,那么仍然是一个不完整的设计,必须要使用abstract来修饰

f). 接口的作用 限定方法的名称 统一标准

类型约束

限定函数或者是成员方法的参数类型

注意: 类型约束 只支持两种类型的限定 一种就是数组,另一种就是对象

多态性

同一个操作 针对不同的子类对象,将产生不同的结果

指在父类中定义的属性或行为被子类继承之后，可以具有不同的数据类型或表现出不同的行为. 这使得同一个属性或行为在父类及其各个子类中具有不同的语义。
